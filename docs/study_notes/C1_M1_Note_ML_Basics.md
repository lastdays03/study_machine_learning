# 1주차 Session 1: 머신러닝 기초와 용어

> **목표**: 머신러닝이 무엇인지 이해하고, 이를 구성하는 핵심 용어들을 명확히 정의합니다.

## 1. 머신러닝이란? (Machine Learning vs Traditional Programming)

### 전통적 프로그래밍 (Traditional Programming)
*   **방식**: 사람이 **규칙(Rule)**과 **데이터(Data)**를 입력하면, 컴퓨터가 **정답(Answer)**을 계산합니다.
*   **예시**: 스팸 메일 필터링을 위해 *"제목에 '광고'가 들어 있다면 스팸함으로 보내라"*는 `if`문을 사람이 직접 코딩합니다.
*   **한계**: 규칙이 너무 복잡하거나(이미지 인식), 규칙을 명확히 정의하기 어려운 경우(자연어 처리) 해결이 어렵습니다.

### 머신러닝 (Machine Learning)
*   **방식**: 사람이 **데이터(Data)**와 **정답(Answer)**을 주면, 컴퓨터가 스스로 **규칙(Rule)**을 찾아냅니다.
*   **예시**: 수만 개의 스팸 메일과 일반 메일을 보여주면, 컴퓨터가 *"이런 단어 패턴이 있으면 스팸일 확률이 높네?"*라고 스스로 판단 기준을 학습합니다.
*   **핵심**: 데이터를 통한 **패턴 학습**입니다.

---

## 2. 머신러닝의 종류 (Types of ML)

### 지도 학습 (Supervised Learning)
*   **특징**: **정답(Label)**이 있는 데이터로 학습합니다. "문제집"과 "해답지"를 주고 공부시키는 것과 같습니다.
*   **대표 문제**: 
    *   **분류(Classification)**: 개 vs 고양이, 스팸 vs 정상 (Category 예측)
    *   **회귀(Regression)**: 아파트 가격 예측, 내일의 기온 예측 (숫자 예측)

### 비지도 학습 (Unsupervised Learning)
*   **특징**: **정답(Label)**이 없습니다. 데이터 자체의 특성이나 구조를 파악하라고 시킵니다.
*   **대표 문제**:
    *   **군집화(Clustering)**: 비슷한 고객끼리 그룹 묶기.
    *   **차원 축소**: 복잡한 데이터를 단순화하기.

### 강화 학습 (Reinforcement Learning)
*   **특징**: 행동에 대한 **보상(Reward)**과 **벌(Penalty)**을 통해 학습합니다. 시행착오를 겪으며 성장합니다.
*   **대표 문제**: 알파고(바둑), 게임 플레이 AI, 로봇 제어.

---

## 3. 핵심 용어 정리 (Key Terms)

데이터를 엑셀 표(Table)라고 생각하면 이해하기 쉽습니다.

| 키(cm) [Feature] | 몸무게(kg) [Feature] | 성별 [Label] |
| :--------------: | :------------------: | :----------: |
|       175        |          70          |     남성     |
|       160        |          50          |     여성     |

### 1) 특성 (Feature)
*   데이터의 **입력 변수**입니다. 예측을 위해 사용하는 단서들입니다.
*   위 표에서는 `키`, `몸무게`가 Feature입니다.
*   수학적으로는 보통 $x$로 표기합니다.

### 2) 레이블 (Label / Target)
*   우리가 예측하고자 하는 **정답**입니다.
*   위 표에서는 `성별`이 Label입니다.
*   수학적으로는 보통 $y$로 표기합니다. (비지도 학습에는 이게 없습니다)

### 3) 훈련 세트 (Training Set) vs 테스트 세트 (Test Set)
*   **Training Set**: 모델을 공부시킬 때 사용하는 데이터입니다. (교과서)
*   **Test Set**: 공부가 잘 되었는지 평가할 때 사용하는 데이터입니다. (수능 시험지)
*   **주의**: 수능 시험지(Test Set)를 미리 보고 공부하면(Training에 포함시키면) 실력을 제대로 평가할 수 없습니다. 이를 **Data Leakage**라고 합니다.

### 4) 과적합 (Overfitting)
*   **상황**: 모델이 **훈련 데이터(Training Set)에는 너무 완벽하게 맞춰졌지만**, 새로운 데이터(Test Set)에는 엉망인 상태입니다.
*   **비유**: 문제집의 답을 달달 외워서 그 문제집은 100점 맞는데, 숫자만 바꿔서 문제를 내면 하나도 못 푸는 학생.
*   **원인**: 학습을 너무 과도하게 했거나, 데이터의 잡음(Noise)까지 패턴으로 외워버린 경우.
